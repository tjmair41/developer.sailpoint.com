IdentityNow Rule Validator

info

The IDN Rule Validator is built and maintained by SailPoint Professional Services. If you have any issues, bugs, or feature requests, please submit them here

Overviewâ€‹

The IdentityNow (IDN) Rule Validator is a simple tool you can use to validate IDN rules for malformed or incorrect code fragments and help make sure they conform to the SailPoint IdentityNow Rule Guide before rule submission. Included is a BeanShell Linter that will take over much of the validation role of rules. It will validate syntax of the BeanShell code and object usage. This isn't designed to replace any sort of unit testing you might do outside IDN. Running the IDN Rule Validator against your rules before submission helps you find problems early on. It is designed to help catch common items that often trip up rule reviews and provide immediate feedback during the rule writing process. However this is not a guarantee that the rule will be approved or that they won't have any other flaws.

Requirementsâ€‹

This utility is a Java based application and requires Java SE Development Kit (JDK) 11 or higher to run.

info

Note that the previous version used JDK 8 (1.8). If you do not have the JDK 11 or above, then you must download and install JDK 11 first.

Support / Featuresâ€‹

The IDN Rule Validator is built and maintained by SailPoint Professional Services. If you have any issues, bugs, or feature requests, please submit them here

Installation and Updatesâ€‹

To use the Rule Validator locally, decompress the sailpoint-saas-rule-validator-{3.0.xx}-distribution.zip package into a folder on your workstation. The sp-rv command executes the java package, and has been designed to be run from a path so can be executed anywhere from your file system.

Linux and MacOS Considerationsâ€‹

Under Linux and MacOS ensure the sp-rv script has execute privileges. To do so, execute the chmod command on the sp-rv script:

chmod +x sp-rv

Run the Rule Validatorâ€‹

To run the validator with standard reporting, invoke this command:

sp-rv --file {path or file name}

This automatically validates all XML files in the path or a specific file name. To automatically include all nested sub-directories in the validation, include the --recursive flag.

sp-rv --file "~/path-to-rules" --recursive
info

If you provide a specific file name, then the --recursive flag parameter is ignored - the validator will only conduct the recursive evaluation if the filePath parameter is null or a directory.

Watch Optionâ€‹

The new rule validator contains a watch feature to monitor a folder for changes.

sp-rv --watch {path}

This will automatically validate all XML files within the specified path. The path must always be a directory and not a file. To automatically include all nested sub-directories in the validation, include the --recursive flag.

sp-rv --watch "~/path-to-rules" --recursive
info

To exit watch mode, press CTRL+C.
